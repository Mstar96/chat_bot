但是现在后端 /chat 接口每次只接收 message 字段，没有接收对话历史，导致模型无法“记住”之前聊过什么。

*前端虽然用 st.session_state 保存了历史消息，但每次请求只发送当前输入，历史被丢弃

改进：

前端把完整的对话历史（或至少最近 N 轮）发送给后端。后端接收历史，构造 messages 列表传给模型，让模型就能根据上下文回答

方法:

1.前端把整个session_state.message全发给后端，后端直接用这个作为message-长对话浪费token

2.会话管理：前端每次请求带一个 session_id，后端用字典存储每个 session_id 的对话历史。后端从存储中取出历史，加上新消息，调用模型，再把新回复存入历史
